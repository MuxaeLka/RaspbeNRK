# NRK Manager

Десктопное приложение на PyQt6 для управления Raspberry Pi 5 через WireGuard VPN.

---

## Возможности

- Мониторинг доступности устройств каждые 5 секунд (TCP-connect на порт 8080)
- Цветовая индикация: зелёный (онлайн) / красный (офлайн)
- Двойной клик по устройству — открывает `http://IP:8080` в браузере
- Кнопки: Открыть, Открыть все, Обновить
- Добавление, редактирование и удаление устройств
- Поиск по имени и IP
- Сортировка по любой колонке
- Журнал событий с временными метками
- Сохранение настроек в `config.json` (рядом с EXE)

---

## Быстрый старт

### 1. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2. Запуск из исходников

```bash
python main.py
```

> `config.json` создаётся автоматически при первом запуске.

---

## Сборка EXE (Windows)

### Установка PyInstaller

```bash
pip install pyinstaller
```

### Команда сборки (один файл)

```bash
pyinstaller --onefile --windowed --name "NRK Manager" main.py
```

### С иконкой (если есть `.ico`-файл)

```bash
pyinstaller --onefile --windowed --icon=resources/icon.ico --name "NRK Manager" main.py
```

### Результат

```
dist/
  NRK Manager.exe   ← готовый исполняемый файл
```

> **Важно:** `config.json` должен лежать в одной папке с `NRK Manager.exe`.
> При первом запуске файл создаётся автоматически с устройствами по умолчанию.

## Конфигурация

`config.json` — простой JSON с массивом устройств:

```json
{
  "devices": [
    {"name": "NRK-1", "ip": "10.60.93.50"},
    {"name": "NRK-2", "ip": "10.60.93.51"}
  ]
}
```

Файл редактируется как через интерфейс приложения, так и вручную.

## Как работает проверка доступности

Вместо ICMP ping используется **TCP-connect** на порт 8080 — это:
- не требует прав администратора
- проверяет именно доступность веб-сервиса, а не просто хоста
- работает через WireGuard без дополнительных настроек

Каждое устройство проверяется в **отдельном потоке** (QThread), интерфейс не блокируется.
